Testing Responsiveness with Chrome DevTools

Steps to check any page's responsiveness:

  1. Right-click the page and choose Inspect (or press F12).
  2. Click the Toggle device toolbar icon (or Ctrl+Shift+M).
  3. Select a device preset (e.g. iPhone, iPad) or drag to resize manually.
  4. Check that text stays readable, nothing overflows horizontally, and buttons/links remain tappable at small sizes.

This page includes <meta name="viewport" content="width=device-width, initial-scale=1.0"> so it scales correctly on mobile — try resizing this browser window to test it.

HTML Validation Using the W3C Validator

The W3C Markup Validator checks a document's HTML against the official spec and flags errors like unclosed tags, missing attributes, or invalid nesting.

Practical: Before and After Validation

✗ Before — contains errors:

<img src="logo.png"> <p>Welcome to my site <h1>Missing closing tags and no alt text</h3>

✓ After — corrected:

<img src="logo.png" alt="Site logo"> <p>Welcome to my site</p> <h1>Fixed: matching closing tag</h1>

Errors fixed: added missing alt attribute, closed the <p> tag, and corrected the mismatched <h1>…</h3> closing tag.

Professionalism Note

No validation errors = publish-ready code. Clean, valid HTML is more predictable across browsers, easier for teammates to maintain, and signals attention to detail — always run new pages through the validator before considering them done.
