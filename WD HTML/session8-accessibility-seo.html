Session 8 — Accessibility & SEO-Friendly HTML

Role of alt, title, and aria-label

  • alt="..." on images — describes the image for screen readers and search engines
  • title="..." — shows a tooltip on hover, adds extra context
  • aria-label="..." — gives assistive technology a label when there's no visible text, e.g. an icon-only button
<img src="logo.png" alt="Company logo">
<a href="/report.pdf" title="Opens a PDF, 2MB">Download Report</a>
<button aria-label="Close menu">×</button>

Meta Tags: description, keywords, viewport

This page's own <head> demonstrates all three — view the page source to see them:

<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="description" content="A short summary shown in search results.">
<meta name="keywords" content="HTML, accessibility, SEO">
  • viewport — makes the page render correctly on mobile devices
  • description — the snippet Google shows under your page title in search results
  • keywords — largely ignored by modern Google, but still used by some search engines

Proper Heading Structure for Search Engines

Headings should form a logical outline — one <h1> per page, followed by <h2> for main sections and <h3> for sub-sections, never skipping levels just for styling.

<h1>Page Title</h1>
  <h2>Section</h2>
    <h3>Sub-section</h3>
  <h2>Another Section</h2>
